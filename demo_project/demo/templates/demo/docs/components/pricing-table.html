Pricing Table

A comprehensive pricing comparison table component for displaying multiple pricing tiers with detailed feature comparisons. Automatically responsive with desktop table and mobile card layouts. 

<c-pricing_table :tiers="[
    {
        'id': 'basic',
        'title': 'Basic',
        'price': '€9',
        'billing_cycle': '/month',
        'button_label': 'Get Started'
    }
]" :sections="[
    {
        'title': 'Features',
        'features': [
            {
                'title': 'Projects',
                'tiers': {'basic': '3', 'pro': '10'}
            }
        ]
    }
]"></c-pricing_table>

Component API

Main Props

Property Type Default Description
tiers list - Array of pricing tier objects (required)
sections list - Array of feature section objects (required)
responsive string true Enable responsive layout (table on desktop, cards on mobile)

Tier Object Structure

Property Type Required Description
id string Yes Unique identifier for the tier
title string Yes Tier name/title
description string No Short description
price string Yes Price display
billing_cycle string No Billing period (e.g., "/month")
button_label string No Call-to-action button text
highlight boolean No Highlight this tier visually
badge string No Badge text (e.g., "Popular")

Section Object Structure

Property Type Description
title string Section heading
features list Array of feature objects in this section

Feature Object Structure

{
  'title': 'Feature Name',
  'tiers': {
    'tier_id': 'value', # String, boolean, or number
    'basic': '5', # String value
    'pro': True, # Boolean (shows checkmark/X)
    'enterprise': 100 # Numeric value
  }
}

Examples

Full Featured Table

A comprehensive pricing table with multiple sections and different value types. 

<c-pricing_table :tiers="[...]" :sections="[...]"></c-pricing_table>

Always Table Layout

Force table layout on all screen sizes by setting responsive="false". 

<c-pricing_table responsive="false" :tiers="[...]" :sections="[...]"></c-pricing_table>

Usage Notes

Responsive Design

On desktop, displays as a comparison table. On mobile, switches to stacked individual pricing plan cards for better readability.

Feature Value Types

Boolean: True shows ✓, False shows ✗
String/Number: Displays the value as-is (e.g., "10GB", "Unlimited", "24/7")

Data Structure

Always use :tiers and :sections (with colons) when passing data structures. The tier id must match keys in feature tiers objects.
