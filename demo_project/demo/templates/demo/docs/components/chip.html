Chip

A versatile chip wrapper component that displays numeric indicators, status badges, or notifications positioned around other elements. Includes optional tooltip support for enhanced user experience.

Messages 
<c-chip text="5" variant="primary">
    <c-button variant="default">Messages</c-button>
</c-chip>

Component API

Property Type Default Description
text string - Text content to display in the chip (required)
position string top-right Chip position: top-right, top-left, bottom-right, bottom-left
variant string primary Color variant: primary, secondary, accent, success, warning, error, info, neutral
size string md Chip size: xs, sm, md, lg, xl
show string true Show/hide the chip (use "false" to hide)
tooltip string - Tooltip text to show on hover
tooltip_position string top Tooltip position: top, bottom, left, right
tooltip_variant string - Tooltip color variant: primary, secondary, accent, info, success, warning, error
class string - Additional CSS classes for the chip

Examples

Basic Usage

Wrap any element with a chip to add numeric indicators or status badges.

Notifications Messages Feature 
<c-chip text="3" variant="error">
    <c-button variant="default" icon_left="bell">Notifications</c-button>
</c-chip>

<c-chip text="12" variant="primary">
    <c-button variant="ghost" icon_left="mail">Messages</c-button>

With Tooltips

Add tooltips to provide additional context when hovering over the wrapped element.

Messages Alerts Features 
<c-chip text="5" variant="primary" 
        tooltip="You have 5 unread messages" 
        tooltip_position="top">
    <c-button variant="default" icon_left="mail">Messages</c-button>
</c-chip>

<c-chip text="!" variant="warning" 
        tooltip="Action required" 
        tooltip_position="bottom" 
        tooltip_variant="warning">
    <c-button variant="default">Alerts</c-button>
</c-chip>

Usage Guidelines

DaisyUI Native Components

This component uses DaisyUI's native Indicator and Badge components for optimal performance and consistent styling.

Best Practices

• Use tooltips for additional context
• Keep chip text short (numbers, single words)
• Choose colors that match the significance (error for urgent notifications)

Accessibility

Consider using tooltips to provide screen reader accessible descriptions of what the chip represents.
