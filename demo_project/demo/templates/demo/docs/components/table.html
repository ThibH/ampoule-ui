Table

A flexible table component with clean syntax for displaying structured data.

Name Email Department Sarah Connor sarah.connor@company.com Engineering John Smith john.smith@company.com Design 
<c-table variant="zebra">
    <c-table.columns>
        <c-table.column>Name</c-table.column>
        <c-table.column>Email</c-table.column>
        <c-table.column>Department</c-table.column>
    </c-table.columns>
    <c-table.rows>
        <c-table.row>
            <c-table.cell>Sarah Connor</c-table.cell>
            <c-table.cell>sarah.connor@company.com</c-table.cell>
            <c-table.cell>Engineering</c-table.cell>
        </c-table.row>
    </c-table.rows>
</c-table>

Component API

c-table

Property Type Default Description
variant string default Table style: default, zebra, pin-rows, pin-cols
filterable boolean false Add a search input to filter table content

c-table.column

Property Type Default Description
sortable boolean false Enable click to sort functionality
sorted boolean false Indicates if this column is currently sorted
direction string asc Sort direction: asc (ascending) or desc (descending)
sort_key string - Data attribute key for custom sorting (optional)
align string left Text alignment: left, center, right

c-table.cell

Property Type Default Description
align string left Text alignment: left, center, right

Table Filters

Les filtres de table utilisent maintenant les composants universels c-filter-options et c-filter-range dans des dropdowns.

Filtre dropdown

<c-dropdown label="Status">
    <c-filter-options filter-id="status" />
</c-dropdown>

Filtre range

<c-dropdown label="Salaire" content_class="w-64">
    <c-filter-range filter-id="salary" format="currency" />
</c-dropdown>

Key Features

  • Clean syntax: Semantic component structure with c-table.columns, c-table.rows, etc.
  • Interactive sorting: Click to sort with automatic direction toggling and visual indicators
  • Flexible sort states: Columns can be sortable, pre-sorted, or both independently
  • Smart value detection: Automatically handles numbers, currency, dates, and custom sort values
  • Universal filtering: Uses the same filter components (c-filter-options, c-filter-range) as everywhere else in the system for consistency
  • Advanced filtering: Combine text search with dropdown and range filters for precise data filtering
  • Auto-configuration: Automatically detects filterable columns and available filter values
  • Multiple filter types: Support for dropdown (checkbox) and range (min/max) filters
  • Intelligent data extraction: Smart detection of filter values from badges, currency, and text content
  • DaisyUI integration: Built on DaisyUI table classes for consistent theming
  • Responsive design: Automatic horizontal scrolling on smaller screens
  • Flexible alignment: Left, center, and right alignment options for columns and cells
  • Rich content support: Cells can contain complex HTML, components, and custom layouts
  • Multiple variants: Default, zebra, compact styles available
  • Django integration: Works seamlessly with Django template loops and filters
