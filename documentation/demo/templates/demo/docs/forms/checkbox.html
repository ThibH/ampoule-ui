Checkbox

Interactive checkbox component with multiple variants and automatic label interactions using modern CSS. 

<c-checkbox id="basic-example" label="I agree to the terms and conditions" />

Component API

Property Type Default Description
id string - Unique identifier for the checkbox input
variant string default Checkbox style: default, pill, card, button
size string md Size: xs, sm, md, lg (default variant only)
label string - Label text
description string - Description text
icon string - Icon name (uses c-icon component)
color string primary Color variant for pill and button styles: primary, secondary, accent, neutral, error, warning, success, info
checked boolean false Initial checked state
disabled boolean false Disabled state
name string - Form input name
value string - Form input value

Variants

Default

Basic checkbox with label and optional description. Click anywhere on the label to toggle.

<c-checkbox id="terms" label="I agree to the terms and conditions" />
<c-checkbox id="newsletter" label="Subscribe to newsletter" description="Get weekly updates" />
<c-checkbox id="notifications" label="Enable notifications" icon="bell" />
<c-checkbox id="disabled-option" label="Disabled option" disabled="true" />

Pills

Compact pill-style checkboxes, perfect for tags or categories. Click anywhere on the pill to toggle.

<c-checkbox id="pill-design" variant="pill" label="Design" />

<c-checkbox id="pill-marketing" variant="pill" label="Marketing" color="secondary" />
<c-checkbox id="pill-sales" variant="pill" label="Sales" icon="trending-up" color="success" />

Key Features

  • Automatic label interaction: Click anywhere on labels, descriptions, pills, cards, or buttons to toggle
  • CSS-only interactivity: Uses modern CSS :has() pseudo-class, no JavaScript required
  • DaisyUI integration: Built on DaisyUI checkbox classes for consistent theming
  • Custom content: Card variant supports custom content via slots
  • Responsive design: Works seamlessly across all screen sizes
  • Unique IDs: Support for unique identifiers to avoid HTML validation errors
