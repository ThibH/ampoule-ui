Radio

Interactive radio button component with multiple variants for mutually exclusive selections. 

<c-radio id="option1" name="group" value="option1" label="Option 1" checked="true" />
<c-radio id="option2" name="group" value="option2" label="Option 2" />
<c-radio id="option3" name="group" value="option3" label="Option 3" />

Component API

Property Type Default Description
id string - Unique identifier for the radio input
name string - Form input name (required to group radios)
value string - Form input value
variant string default Radio style: default, pill, card, button
size string md Size: xs, sm, md, lg (default variant only)
label string - Label text
description string - Description text
icon string - Icon name (uses c-icon component)
color string primary Color variant for pill and button styles: primary, secondary, accent, neutral, error, warning, success, info
checked boolean false Initial checked state
disabled boolean false Disabled state

Variants

Default

Basic radio buttons with labels and optional descriptions. Click anywhere on the label to select.

<c-radio id="plan-free" name="plan" value="free" label="Free Plan" description="Basic features" />
<c-radio id="plan-pro" name="plan" value="pro" label="Pro Plan" checked="true" />
<c-radio id="plan-enterprise" name="plan" value="enterprise" label="Enterprise Plan" icon="building" />

Pills

Compact pill-style radio buttons, perfect for categories or options. Click anywhere on the pill to select.

<c-radio id="pill-xs" name="size-group" value="xs" variant="pill" label="XS" />
<c-radio id="pill-sm" name="size-group" value="sm" variant="pill" label="SM" checked="true" />
<c-radio id="pill-md" name="size-group" value="md" variant="pill" label="MD" color="secondary" />

Cards

Card-style radio buttons with rich content support. Click anywhere on the card to select.

    value="starter"
    variant="card"

Key Features

  • Mutually exclusive selection: Only one option can be selected per group (defined by name attribute)
  • Automatic label interaction: Click anywhere on labels, descriptions, pills, cards, or buttons to select
  • CSS-only interactivity: Uses modern CSS :has() pseudo-class, no JavaScript required
  • DaisyUI integration: Built on DaisyUI radio classes for consistent theming
  • Custom content: Card variant supports custom content via slots
  • Responsive design: Works seamlessly across all screen sizes
  • Grouped selections: Use the same name attribute to group radio buttons together
