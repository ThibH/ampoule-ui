Input

A comprehensive input component with support for labels, validation, icons, and various input types. Includes Django forms integration. 

<c-form.input type="text" placeholder="Enter your text..." />

Component API

Property Type Default Description
field Django Field - Django form field for automatic integration
type string text Input type: text, email, password, url, tel, search, number
placeholder string - Placeholder text for the input
label string - Label text displayed above the input
description string - Help text displayed below the input
icon_leading string - Icon name to display on the left side
icon_trailing string - Icon name to display on the right side
text_leading string - Text to display on the left side (prefix)
text_trailing string - Text to display on the right side (suffix)
size string md Input size: xs, sm, md, lg
variant string default Color variant: default, primary, secondary, accent, info, success, warning, error
disabled boolean false Disable the input
readonly boolean false Make the input read-only
required boolean false Make the input required
invalid boolean false Mark the input as invalid/error state
clearable boolean false Add a clear button to empty the input
id string - Unique identifier for the input
name string - Form field name
value string - Default or current value

Django Form Integration

Using with Django Forms

The component automatically integrates with Django form fields when you pass the field parameter.

# In your template:
<c-form.input field="{{ form.email }}" label="Email Address" icon_leading="mail" />
<c-form.input field="{{ form.password }}" label="Password" icon_leading="lock" /> 
<!-- Django form integration -->
<c-form.input field="{{ form.email }}" label="Email Address" icon_leading="mail" />
<c-form.input field="{{ form.password }}" label="Password" icon_leading="lock" />

<!-- The component will automatically: -->
<!-- - Set name, id, value from the field -->
<!-- - Display validation errors -->
<!-- - Handle required state -->

Automatic Integration

When using the field parameter, the component automatically extracts the field name, value, and validation state from the Django form field.

Error Handling

Form validation errors are automatically displayed below the input with proper styling.

Accessibility

Always provide proper labels for better accessibility. The component automatically links labels to inputs when an ID is provided.
