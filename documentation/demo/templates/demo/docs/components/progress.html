Progress

A progress bar component for displaying task completion, loading states, and value indicators with DaisyUI styling. 

<c-progress value="50" max="100" />

Component API

Property Type Default Description
variant string neutral Color variant (primary, secondary, accent, neutral, info, success, warning, error)
value string - Current progress value
max string 100 Maximum value for the progress bar
show_label boolean false Whether to display the progress value as text
label string - Custom label text to display (overrides automatic value display)
title string - Title to display above the progress bar
indeterminate boolean false Whether to show an indeterminate (loading) progress bar
class string - Additional CSS classes for the progress element

Examples

Color Variants

Choose from different color themes to match your design and convey meaning. 

<!-- Primary variant -->
<c-progress variant="primary" value="60" />

<!-- Secondary variant -->
<c-progress variant="secondary" value="40" />

<!-- Success variant -->
<c-progress variant="success" value="90" />

<!-- Error variant -->
<c-progress variant="error" value="30" />

With Labels

Display progress values as text labels for better user feedback. 

<!-- Percentage display -->
<c-progress variant="primary" value="75" show_label="true" />

<!-- Custom max value -->
<c-progress variant="success" value="20" max="50" show_label="true" />

<!-- Custom label -->
<c-progress variant="info" value="45" show_label="true" label="53.0g" />

<!-- Fraction display -->
<c-progress variant="warning" value="15" max="25" show_label="true" />

With Titles

Add descriptive titles above progress bars for better context. 

<!-- With title and percentage -->
<c-progress variant="primary" value="65" title="Download Progress" show_label="true" />

<!-- With title and custom label -->
<c-progress variant="warning" value="45" title="Memory Usage" show_label="true" label="4.2GB / 8GB" />

<!-- With title and custom max -->
<c-progress variant="info" value="30" title="Storage Used" show_label="true" max="500" label="150GB" />

Indeterminate Progress

Use indeterminate progress bars for loading states when the completion time is unknown.

Please wait while we process your files... Syncing with server... 
<!-- Basic indeterminate -->
<c-progress variant="primary" indeterminate="true" />

<!-- With title -->
<c-progress variant="success" title="Uploading Files" indeterminate="true">
    Please wait while we process your files...
</c-progress>

<!-- With custom message -->
<c-progress variant="info" title="Processing Data" indeterminate="true" />

Different Value Formats

Examples of various ways to display progress values with custom labels. 

<!-- Percentage format -->
<c-progress variant="primary" value="75" title="Percentage" show_label="true" />

<!-- Rating format -->
<c-progress variant="success" value="8" max="10" title="Rating" show_label="true" />

<!-- Weight format -->
<c-progress variant="info" value="45" title="Weight" show_label="true" label="53.0g" />

<!-- Budget format -->
<c-progress variant="warning" value="1500" max="2000" title="Budget Used" show_label="true" label="$1,500 / $2,000" />

<!-- Tasks format -->
<c-progress variant="neutral" value="7" max="10" title="Progress" show_label="true" label="7 of 10 tasks" />

With Additional Content

Add extra information below the progress bar using slot content.

Uploading document.pdf - 5 minutes remaining Installing dependencies... Almost done! High memory usage detected. Consider closing some applications. Running code analysis and security checks... 
<!-- With upload info -->
<c-progress variant="primary" value="45" title="File Upload" show_label="true">
    Uploading document.pdf - 5 minutes remaining
</c-progress>

<!-- With installation message -->
<c-progress variant="success" value="90" title="Installation" show_label="true">
    Installing dependencies... Almost done!
</c-progress>

<!-- With warning message -->
<c-progress variant="warning" value="75" title="Memory Usage" show_label="true" label="6GB / 8GB">
    High memory usage detected. Consider closing some applications.
</c-progress>

Appendix

Usage Guidelines

Color Meaning

Use semantic colors: success for completion, warning for moderate levels, error for critical states, and info for general progress.

Indeterminate Usage

Use indeterminate progress only when you cannot predict completion time. Provide context through titles or slot content.

Accessibility

Always provide descriptive titles and use aria-labels when necessary. Consider screen reader users when displaying progress information.
